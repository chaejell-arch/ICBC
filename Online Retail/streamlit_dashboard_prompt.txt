**프롬프트:**

`Online Retail` 데이터셋을 기반으로 상호작용 가능한 Streamlit 대시보드를 생성해 주세요.

**대시보드 목표:**
`Online Retail/eda.py` 파일에서 수행된 탐색적 데이터 분석(EDA) 결과를 체계적이고 탐색하기 쉬운 형태로 시각화하여 온라인 소매 데이터에 대한 깊이 있는 통찰력을 제공합니다.

**1. 데이터 로드 및 전처리:**
*   `Online Retail/Online_Retail.csv` 파일을 로드하고 결과를 캐시(`@st.cache_data`)하여 성능을 최적화합니다.
*   `eda.py` 파일에 정의된 다음 전처리 단계를 적용합니다:
    *   결측치 제거 (특히 `CustomerID` 기준).
    *   `Description` 컬럼의 결측치 `알 수 없음`으로 대체.
    *   `CustomerID`를 정수형으로, `InvoiceDate`를 datetime 객체로 변환.
    *   취소 주문 (InvoiceNo가 'C'로 시작) 제거.
    *   `Quantity`와 `UnitPrice`가 0보다 작은 데이터 제거.
    *   `TotalPrice` (Quantity * UnitPrice) 컬럼 생성.

**2. 대시보드 레이아웃 및 메뉴:**
*   **사이드바 메뉴:** `st.sidebar`를 사용하여 다음 메뉴를 구성합니다. 사용자가 메뉴를 선택하면 해당 페이지가 표시되어야 합니다.
    *   데이터 개요
    *   데이터 검색 기능
    *   판매 분석
    *   상품 분석
    *   고객 분석
    *   ARPU 분석
*   **탭(Tabs) 활용:** 각 분석 페이지 내에서는 `st.tabs`를 사용하여 관련 시각화들을 주제별로 묶어 명확하게 분리합니다.

**3. 데이터 개요 페이지:**
*   전처리된 데이터의 기본 정보를 다음 탭으로 나누어 표시합니다:
    *   **데이터 샘플:** `st.dataframe(df.head())`
    *   **데이터 정보:** `st.text(df.info())`의 요약본
    *   **기술 통계량:** `st.dataframe(df.describe())`

**4. 데이터 검색 기능 페이지:**
*   사용자가 원본 데이터를 동적으로 필터링하고 검색할 수 있는 인터페이스를 제공합니다.
*   **필터링 옵션:**
    *   **국가(`Country`):** `st.multiselect`를 사용하여 여러 국가를 선택할 수 있도록 합니다.
    *   **상품 설명(`Description`):** `st.text_input`을 사용하여 키워드로 상품 설명을 검색할 수 있도록 합니다.
    *   **총액(`TotalPrice`):** `st.slider`를 사용하여 총액 범위를 지정할 수 있도록 합니다.
    *   **날짜 범위(`InvoiceDate`):** `st.date_input`을 사용하여 조회할 날짜 범위를 선택할 수 있도록 합니다.
*   **결과 표시:** 필터링된 결과를 `st.dataframe`을 통해 표시하고, 검색된 데이터의 행 수를 함께 보여줍니다.

**5. 판매 분석 페이지:**
*   `st.tabs`를 사용하여 다음 분석 내용을 분리합니다:
    *   **"시간/요일별 분석" 탭:**
        *   시간대별 주문 수 (Plotly 막대 그래프)
        *   요일별 주문 수 (Plotly 막대 그래프)
    *   **"국가별/매출 추이" 탭:**
        *   상위 10개 국가별 주문 수 (Plotly 막대 그래프)
        *   월별 총 매출 추이 (Plotly 선 그래프)

**6. 상품 분석 페이지:**
*   `st.tabs`를 사용하여 다음 분석 내용을 분리합니다:
    *   **"판매 순위" 탭:**
        *   상위 10개 판매 상품 (수량 기준, Plotly 가로 막대 그래프)
        *   상위 10개 매출 상품 (금액 기준, Plotly 가로 막대 그래프)
    *   **"가격/수량 관계" 탭:**
        *   단가와 수량의 관계 (Plotly 산점도, 로그 스케일 적용)

**7. 고객 분석 페이지:**
*   `st.tabs`를 사용하여 다음 분석 내용을 분리합니다:
    *   **"구매 패턴" 탭:**
        *   고객별 총 구매액 분포 (Plotly 히스토그램)
        *   고객별 주문 빈도 분포 (Plotly 히스토그램)
    *   **"RFM 분석" 탭:**
        *   `eda.py`의 RFM 분석 로직을 적용하여 고객 세그먼트 분포를 시각화 (Plotly 가로 막대 그래프)
        *   각 RFM 세그먼트에 대한 설명을 `st.expander` 내에 제공합니다.

**8. ARPU 분석 페이지:**
*   `st.tabs`를 사용하여 다음 분석 내용을 분리합니다:
    *   **"ARPU 추이" 탭:**
        *   월별 ARPU 추이 (Plotly 선 그래프 및 막대 그래프)
    *   **"데이터 테이블" 탭:**
        *   월별 ARPU 상세 데이터 (총 매출, 구매 고객 수, ARPU)를 `st.dataframe`으로 표시합니다.

**제약 사항:**
*   모든 데이터 시각화는 `Plotly` 라이브러리만을 사용하여 구현해야 합니다.
*   `matplotlib` 등 다른 시각화 라이브러리는 사용하지 않습니다.
*   코드를 직접 작성하지 않습니다. 위에 명시된 기능들을 Streamlit과 Plotly를 사용하여 구현하도록 지시하는 형태의 프롬프트만 제공합니다.
