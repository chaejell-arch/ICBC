# 범용 데이터 분석 워크플로우

이 문서는 어떤 데이터가 주어지더라도 체계적으로 분석을 수행할 수 있도록 돕는 범용 데이터 분석 워크플로우를 제공합니다. 각 단계는 일반적인 분석 절차를 따르며, 필요에 따라 특정 단계를 건너뛰거나 순서를 변경할 수 있습니다.

## 1. 데이터 준비 (Data Preparation)

분석을 시작하기 전, 데이터를 불러오고 정제하는 과정입니다.

### 1.1. 라이브러리 및 데이터 불러오기

분석에 필요한 라이브러리를 설치하고 임포트한 후, CSV, Excel, DB 등 다양한 소스로부터 데이터를 불러옵니다.

### 1.2. 데이터 기본 정보 확인

데이터의 구조, 타입, 크기, 컬럼 정보 등 기본적인 정보를 파악하여 데이터에 대한 전반적인 이해를 높입니다. 기술 통계량을 통해 수치형 데이터의 분포를 개략적으로 확인합니다.

### 1.3. 결측치 확인 및 처리

데이터 품질을 높이기 위해 비어있는 값(결측치)의 분포와 양을 확인합니다. 확인 후, 해당 행을 제거하거나, 평균, 중앙값, 최빈값 등의 대표값으로 채우는 등 적절한 처리 전략을 결정하고 수행합니다.

## 2. 데이터 탐색 (Exploratory Data Analysis - EDA)

데이터의 패턴, 변수 간의 관계, 이상치 등을 시각화와 통계적 방법을 통해 깊이 있게 탐색합니다.

### 2.1. 수치형 데이터 탐색

- **분포 확인**: 히스토그램, 박스 플롯 등을 사용해 개별 수치형 데이터의 분포, 중심 경향성, 산포도 등을 시각적으로 확인합니다.
- **상관 관계 분석**: 스캐터 플롯(산점도), 히트맵 등을 사용해 여러 수치형 변수들 간의 선형적 또는 비선형적 관계를 분석합니다.

### 2.2. 범주형 데이터 탐색

- **빈도 분석**: 바 차트, 카운트 플롯 등을 사용해 각 범주의 빈도와 비율을 확인하여 데이터의 구성을 파악합니다.

## 3. 데이터 전처리 (Data Preprocessing)

모델링 단계의 성능을 높이기 위해 데이터를 적절한 형태로 가공하고 정제합니다.

### 3.1. 이상치 처리 (Outlier Handling)

EDA 과정에서 발견된 이상치(Outlier)가 분석 결과에 미치는 영향을 고려하여, 이를 제거하거나 다른 값으로 대체하는 등의 처리를 수행합니다.

### 3.2. 특성 공학 (Feature Engineering)

분석 목적에 더 적합한 새로운 변수를 만들기 위해 기존 변수들을 조합하거나 분해하고, 변형하는 작업을 수행합니다. 예를 들어, 날짜 데이터에서 연도, 월, 요일 등의 정보를 추출할 수 있습니다.

### 3.3. 데이터 스케일링

서로 다른 단위를 가진 수치형 변수들의 스케일을 맞추는 작업입니다. 모델이 변수의 크기에 영향을 받지 않도록 표준화(Standardization) 또는 정규화(Normalization) 등의 기법을 적용합니다.

### 3.4. 범주형 변수 인코딩

머신러닝 모델이 이해할 수 있도록, 문자열 형태의 범주형 변수를 숫자 형태의 벡터로 변환합니다. 원-핫 인코딩(One-Hot Encoding)이나 레이블 인코딩(Label Encoding) 등의 기법이 사용됩니다.

## 4. 모델링 (Modeling)

분석 목적(예: 분류, 회귀, 군집화)에 따라 적절한 머신러닝 알고리즘을 선택하고, 전처리된 데이터를 사용하여 모델을 학습시키고 성능을 평가합니다.

- **데이터 분할**: 모델 학습과 평가를 위해 전체 데이터를 학습용(Train Set)과 테스트용(Test Set)으로 분리합니다.
- **모델 학습**: 선택된 알고리즘을 사용하여 학습용 데이터로부터 패턴을 학습시킵니다.
- **모델 평가**: 테스트용 데이터를 사용하여 모델의 예측 성능을 정확도, 정밀도, 재현율, F1 스코어 등 적절한 평가지표로 측정합니다.

## 5. 결론 및 인사이트 도출

모델의 평가 결과와 EDA 과정에서 얻은 분석 결과를 종합하여 최종 결론을 내립니다. 이를 바탕으로 데이터에 숨겨진 의미 있는 정보(인사이트)를 발견하고, 비즈니스 관점에서의 actionable한 제안이나 의사결정에 활용합니다.
